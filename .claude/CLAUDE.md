# Development Workflow Design

**Systematic skill routing for development teams using proven methodologies.**

## Core Philosophy

Most teams work reactively—choosing approaches ad-hoc based on immediate pressure rather than systematic principles. This skill provides a routing system to direct development tasks to the appropriate specialized methodology.

**Key insight:** The goal is not to follow one methodology religiously, but to systematically route the right specialized skill to the right problem type.

## Core Development Principles
- **Surgical approach:** Use `surgical-coding-principles` skill for precise, minimal changes
- **Evaluation-first:** Apply `ai-system-evaluation` skill for any AI/LLM system work  
- **Systematic improvement:** Use `autonomous-improvement-loops` skill for optimization tasks
- **Context awareness:** Apply `context-engineering-patterns` skill for information management

## Skill Routing Framework

### For Complex AI/Agent Development
**Building systems with LLMs, agents, or complex AI workflows**

- **Agentic systems and workflows** → `agentic-development-workflows`
- **LLM system architecture decisions** → `llm-system-design-patterns`  
- **Context and attention management** → `context-engineering-patterns`
- **AI system quality and measurement** → `ai-system-evaluation`

### For Learning & Skill Development
**Rapidly building competence in new domains**

- **Learning new technical areas** → `iterative-learning-approach`
- **Setting up self-improving systems** → `autonomous-improvement-loops`
- **Building development workflows** → `development-workflow-design` (this skill)

### For Quality & Development
**Systematic approaches to quality and precise implementation**

- **Any coding work requiring precision** → `surgical-coding-principles`
- **AI system quality and measurement** → `ai-system-evaluation`
- **Performance optimization tasks** → `autonomous-improvement-loops`

### For System Architecture & Design
**Building reliable, scalable systems**

- **LLM-powered system architecture** → `llm-system-design-patterns`
- **Managing complex information flows** → `context-engineering-patterns`
- **Evaluation and monitoring systems** → `ai-system-evaluation`

## Common Workflow Patterns

### API / Third-Party Integration Development
- Apply `surgical-coding-principles`: minimal, focused changes for all implementation work
- Use `ai-system-evaluation` for API reliability testing
- Apply `context-engineering-patterns` for managing complex queries and payloads

### Data Pipeline & Automation Development  
- Apply `surgical-coding-principles`: precise implementation of data processing logic
- Use `ai-system-evaluation` for processing accuracy
- Apply `agentic-development-workflows` for multi-step workflows
- Use `autonomous-improvement-loops` for reliability optimization

### LLM-Powered Systems (chatbots, agents, natural-language interfaces)
- Apply `surgical-coding-principles`: precise implementation of system components
- Apply `llm-system-design-patterns` for system architecture
- Use `context-engineering-patterns` for natural language query handling  
- Apply `autonomous-improvement-loops` for prompt optimization
- Use `ai-system-evaluation` for quality measurement

### Learning New Technical Domains
- Apply `surgical-coding-principles`: build understanding through precise, small experiments
- Primary: `iterative-learning-approach` for rapid skill development
- Supporting: `agentic-development-workflows` for building while learning
- Quality: `ai-system-evaluation` for measuring learning progress

### Complex System Debugging
- Primary: `surgical-coding-principles` for precise problem isolation and fixes
- Supporting: `ai-system-evaluation` for systematic failure analysis
- Architecture: `llm-system-design-patterns` for understanding system behavior

### Performance Optimization Projects
- Apply `surgical-coding-principles`: make precise, measured changes
- Primary: `autonomous-improvement-loops` for systematic optimization
- Measurement: `ai-system-evaluation` for tracking improvements

## When to Combine Multiple Skills

### High-Complexity Projects
**Use 3+ skills for comprehensive coverage**
- New AI system in unfamiliar domain
- Performance-critical systems requiring optimization
- Legacy system modernization with AI components

### Medium-Complexity Projects  
**Use 2-3 skills for focused approach**
- Extending existing systems with new features
- Debugging complex multi-component issues
- Learning new framework while building

### Simple Projects
**Use 1-2 skills for targeted intervention**
- Bug fixes and small enhancements
- Well-understood feature additions
- Performance optimizations in known systems

## Quality Gates
- [ ] Clear success criteria defined upfront (`ai-system-evaluation`)
- [ ] Systematic evaluation approach in place (`ai-system-evaluation`)
- [ ] Context managed to prevent degradation (`context-engineering-patterns`)
- [ ] Changes made surgically with clear scope (`surgical-coding-principles`)
- [ ] Learning approach planned for new domains (`iterative-learning-approach`)
- [ ] Self-improvement mechanisms in place (`autonomous-improvement-loops`)
- [ ] System architecture patterns followed (`llm-system-design-patterns`)

## Success Metrics

You're applying this routing system effectively when:
- Team members consistently choose appropriate skills for different problem types
- Quality outcomes improve through systematic application of specialized methodologies
- Development velocity increases due to reduced methodology selection overhead
- Complex projects coordinate multiple skills effectively
- Learning and improvement happen systematically rather than ad-hoc
- Architecture decisions follow proven patterns rather than ad-hoc approaches
